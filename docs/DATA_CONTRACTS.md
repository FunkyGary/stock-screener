# Data Contracts

Compact schema notes for agents. Prefer these contracts before opening generated
JSON snapshots.

## Ownership Rules

| File | Source of truth | Commit policy |
|---|---|---|
| `data/watchlist.csv` | Human edited | Commit when the requested change is a watchlist change. |
| `data/sector_map.csv` | `scripts/build_sector_map.py` output | Commit only when intentionally refreshing classifications. |
| `data/latest_signals.json` | Generated by `screener.run` and Actions | Do not commit feature-related snapshot churn unless requested. |
| `data/analyst_target_events.jsonl` | US EOD run merge output | Commit when generated by the intended US data update workflow. |
| `data/tw_target_events.jsonl` | Manual TW target events | Commit intentional manual target-event additions. |

## `watchlist.csv`

Columns:

- `symbol`: yfinance symbol. TWSE examples use `.TW`; TPEx examples may use
  `.TWO`; US symbols have no suffix.
- `market`: `tw` or `us`.
- `name`: dashboard display name.
- `tradingview_symbol`: TradingView embed symbol such as `TWSE:2330`,
  `TPEX:6173`, or `NASDAQ:NVDA`.

When adding symbols, keep `market` lowercase and ensure sector-map coverage if
sector strength matters for the new symbol.

## `sector_map.csv`

Columns:

- `symbol`
- `market`
- `sector_official`
- `industry_group`
- `industry`
- `source`

TW rows come from TWSE/TPEx OpenAPI classifications. US rows come from yfinance
sector/industry data. Sector scoring groups by `industry_group`; changing that
field changes scoring behavior.

## `latest_signals.json`

Top level:

- `generated_at`: UTC ISO timestamp.
- `last_run`: timestamps keyed by `tw`, `tw_eod`, `tw_intraday`, `us`,
  `us_eod`, `us_intraday`.
- `market_regime`: index-level trend context.
- `signals`: object keyed by symbol.

`market_regime.markets.tw.strategy` records the 0050-based TW scoring regime:

- `strategy`: `bear_crash`, `range`, or `bull`
- `label`, `reason`
- 0050 metrics: `close`, `ma20`, `ma60`, `ma240`, `return_60d`,
  `drawdown_120d`
- `thresholds`: currently 120-day drawdown ≤ -12% for bear/crash and
  60-day return > 3% for bull.

Signal record:

- identity: `symbol`, `market`, `name`, `tradingview_symbol`, `status`, `mode`
- score fields: `score`, `max_score`, `reasons`
- data blobs: `indicators`, `analyst`, `chip`, `sector`
- TW scoring context: `score_regime`
- failed fetch: `status: "fetch_failed"` plus `error`

Reason record:

- `rule`
- `passed`
- `detail`
- `weight`
- optional `score` when partial credit differs from full weight

Indicator keys are defined by `screener.indicators.IndicatorSnapshot`. Do not
rename them without updating scoring, dashboard, and tests.

## Target Event JSONL

Each line is one JSON object. Normalization lives in `screener.io`; use those
helpers for merge behavior instead of hand-written dedupe.

Common fields:

- `symbol`
- `market`
- `event_date`
- `published_at`
- `firm`
- `action`: currently `raise`
- `previous_target`
- `target_price`
- `source`
- `event_id`: stable 24-character hash when not supplied

US event history may also include:

- `fetched_at`
- `raise_pct`
- `close_at_fetch`
- `upside_pct`
- `headline`
- `url`

Manual TW events usually need only symbol, market, event date, published time,
firm, action, target price, and source. Keep `published_at` timezone-aware.

## Generated Data Caution

`data/latest_signals.json` can change because prices changed, not because code
changed. For ordinary code or documentation work, leave generated snapshots
unstaged unless the user explicitly asks for refreshed data.
