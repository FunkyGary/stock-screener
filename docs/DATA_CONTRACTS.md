# Data Contracts

Compact schema notes for agents. Prefer these contracts before opening generated
JSON snapshots.

## Ownership Rules

| File | Source of truth | Commit policy |
|---|---|---|
| `data/watchlist.csv` | Human edited | Commit when the requested change is a watchlist change. |
| `data/sector_map.csv` | `scripts/build_sector_map.py` output | Commit only when intentionally refreshing classifications. |
| `data/latest_signals.json` | Generated by `screener.run` and Actions | Do not commit feature-related snapshot churn unless requested. |
| `data/analyst_target_events.jsonl` | US EOD run merge output | Commit when generated by the intended US data update workflow. |
| `data/tw_target_events.jsonl` | Manual TW events + Cnyes FactSet EOD merge output | Commit intentional manual or automated target-event additions. |
| `data/valuation_snapshots.jsonl` | EOD run append output (PE/PB/EPS-surprise) | Commit when generated by EOD data update workflows; append-only daily log. |

## `watchlist.csv`

Columns:

- `symbol`: yfinance symbol. TWSE examples use `.TW`; TPEx examples may use
  `.TWO`; US symbols have no suffix.
- `market`: `tw` or `us`.
- `name`: dashboard display name.
- `tradingview_symbol`: TradingView embed symbol such as `TWSE:2330`,
  `TPEX:6173`, or `NASDAQ:NVDA`.

When adding symbols, keep `market` lowercase and ensure sector-map coverage if
sector strength matters for the new symbol.

## `sector_map.csv`

Columns:

- `symbol`
- `market`
- `sector_official`
- `industry_group`
- `industry`
- `source`

TW rows come from TWSE/TPEx OpenAPI classifications. US rows come from yfinance
sector/industry data. Sector scoring groups by `industry_group`; changing that
field changes scoring behavior.

## `latest_signals.json`

Top level:

- `generated_at`: UTC ISO timestamp.
- `last_run`: timestamps keyed by `tw`, `tw_eod`, `tw_intraday`, `us`,
  `us_eod`, `us_intraday`.
- `market_regime`: index-level trend context.
- `signals`: object keyed by symbol.

`market_regime.markets.<market>.strategy` records the benchmark-based scoring
regime. TW uses 0050; US uses SPY:

- `strategy`: `bear_crash`, `bear_downtrend`, `range`, or `bull`
- `label`, `reason`
- benchmark metrics: `close`, `ma5`, `prev_ma5`, `ma10`, `ma20`, `ma60`,
  `ma240`, `return_60d`, `drawdown_120d`
- `thresholds`: currently 120-day drawdown <= -12% for bear/crash;
  `close < ma240` and `ma60 < ma240` for bear/downtrend; and 60-day return
  > 3% plus the MA conditions for bull.

Signal record:

- identity: `symbol`, `market`, `name`, `tradingview_symbol`, `status`, `mode`
- score fields: `score`, `max_score`, `reasons`
- data blobs: `indicators`, `analyst`, `chip`, `sector`
- regime scoring context: `score_regime`
- failed fetch: `status: "fetch_failed"` plus `error`

Reason record:

- `rule`
- `passed`
- `detail`
- `weight`
- optional `score` when partial credit differs from full weight

Indicator keys are defined by `screener.indicators.IndicatorSnapshot`. Do not
rename them without updating scoring, dashboard, and tests.

Intraday snapshots may include projected-volume keys:

- `projected_volume`: estimated full-day volume from intraday curves.
- `projected_vol_ratio`: projected volume divided by the prior 20 daily-volume
  average, capped at 5x.
- `same_time_vol_ratio`: current cumulative volume divided by the same-time
  historical median.
- `volume_projection_source`: `symbol`, `industry`, or `market`.
- `volume_projection_reliable`: false during the first 30 regular-session
  minutes or when no curve is available.
- `volume_projection_capped`: true when the projected ratio hit the cap.

## Target Event JSONL

Each line is one JSON object. Normalization lives in `screener.io`; use those
helpers for merge behavior instead of hand-written dedupe.

Common fields:

- `symbol`
- `market`
- `event_date`
- `published_at`
- `firm`
- `action`: currently `raise`
- `previous_target`
- `target_price`
- `source`
- `event_id`: stable 24-character hash when not supplied

US event history may also include:

- `fetched_at`
- `raise_pct`
- `close_at_fetch`
- `upside_pct`
- `headline`
- `url`

TW events usually need only symbol, market, event date, published time, firm,
action, target price, and source. Keep `published_at` timezone-aware. Automated
Cnyes FactSet valuation events use `source: "cnyes_factset"`; target-price raise
articles use `action: "raise"`, while EPS-estimate articles that only include a
target price use `action: "update"` so they can be displayed without scoring as
target raises.

## Valuation / Fundamentals (display-only)

Each `latest_signals.json` signal record carries an optional `fundamental` blob.
It is **display-only** and is never passed to `screener.score.score()`:

- `pe`, `pb`: trailing PE / price-to-book.
  - TW: TWSE `BWIBBU_ALL` + TPEx peratio openapi, fetched once per EOD run
    (`fetch.fetch_tw_valuation_map`). PE can be `null` (blank cell upstream).
  - US: yfinance `.info` (`fetch.fetch_us_fundamental`).
- `eps_surprise_pct`, `eps_period`: US only, latest reported quarter from Finnhub
  `company_earnings`.
- `margins`: US only, recent quarterly margins newest-first, up to 6 quarters:
  `[{"period": "YYYY-MM-DD", "gm": <gross%>, "om": <operating%>, "nm": <net%>}]`.
  From the yfinance quarterly income statement (`fetch._margins_from_income_stmt`).
  Any of `gm`/`om`/`nm` may be `null`; quarters with missing/zero revenue are
  dropped. Used by the dashboard for a profitability-trend view and a 本業 vs 業外
  (operating vs net) divergence flag — display-only, never scored.
- `revenues`: US only, recent quarterly Total Revenue newest-first, up to 8
  quarters: `[{"period": "YYYY-MM-DD", "revenue": <float>}]`. From the same
  yfinance quarterly income statement (`fetch._revenues_from_income_stmt`).
- `eps_actuals`: US only, reported quarterly EPS actuals newest-first:
  `[{"period": "YYYY-MM-DD", "eps": <float>}]`. From the same Finnhub
  `company_earnings` payload as the EPS surprise (`fetch._eps_actuals_from_earnings`).
  `revenues` + `eps_actuals` drive the dashboard's 領先佈局 view (營收 YoY, 營收/EPS
  落底回升, and a sell-the-news caution) — display-only, never scored.
- `source`: provenance string (`twse` / `tpex` / `yfinance/finnhub`).

EOD runs also append a point-in-time row per ok signal to
`data/valuation_snapshots.jsonl` (`io.append_valuation_snapshots`, deduped by
`(date, symbol)`). Each row carries `pe`, `pb`, `eps_surprise_pct`, `eps_period`,
and the latest-quarter margins (`margin_period`, `gross_margin`,
`operating_margin`, `net_margin`). This is the forward dataset for a future
fundamentals backtest. Fundamentals are EOD-only; intraday runs carry the
previous `fundamental` blob forward unchanged.

## Generated Data Caution

`data/latest_signals.json` can change because prices changed, not because code
changed. For ordinary code or documentation work, leave generated snapshots
unstaged unless the user explicitly asks for refreshed data.
